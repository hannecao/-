# 章节验收表

每章 raw 初稿必须逐项验收。任一硬性项失败，不得直接标记完成。

## 一、硬格式

- [ ] 人称正确
- [ ] 全程唯一视角，无越肩、无上帝视角
- [ ] 系统或特殊角色台词格式正确
- [ ] 正文没有章节号、模型说明、现实元信息
- [ ] 没有违反年龄数字、禁词、专名规则
- [ ] 时间、地点、空间走位清楚
- [ ] 每场至少一个非视觉感官细节
- [ ] 只输出正文
- [ ] 字数落在规定区间

## 二、章节功能

- [ ] 能用一句话说明本章为何存在
- [ ] 只有一个核心戏剧发动机
- [ ] 主角目标清楚
- [ ] 冲突由前情自然产生
- [ ] 转折是事件、动作、选择或认知变化，不是突然想通
- [ ] 章末钩子由本章事件自然触发
- [ ] 伏笔只写到细纲允许的层级

## 三、事件与因果

- [ ] 事件型章节只有一条乱子主线
- [ ] 因能追溯到前文已经发生的改变
- [ ] 问题升级过程清楚
- [ ] 解决步骤具体、可执行、符合世界规则
- [ ] 没有靠系统总结来强行连接两件不相关的事
- [ ] 没有为了制造戏剧性而临时创造新制度

## 四、人物

- [ ] 主角亲自判断、选择、调查、承担并解决核心问题
- [ ] 男主、配角、系统没有替主角给出完整答案
- [ ] 人物修为、身份、位置、关系正确
- [ ] 人物只知道其当前应知的信息
- [ ] 对话符合人物身份和关系阶段
- [ ] 没有为了展示男主魅力而抢走本章高光
- [ ] 没有把非反派角色写成工具化恶人

## 五、设定与暗线

- [ ] 没有新增未经定案的世界机制
- [ ] 没有新增光环转移、能量转移、数值转接等规则
- [ ] 没有把临时描述写成正式专名
- [ ] 没有堆叠细纲之外的异象
- [ ] 没有让系统直接讲清世界真相
- [ ] 严格区分现象、疑点、猜测、证据、答案
- [ ] 没有提前泄露下一阶段真相

## 六、文风与节奏

- [ ] 吐槽来自人物处境，不是为了完成笑点指标
- [ ] 同一梗没有连续重复
- [ ] 没有每段强行抖包袱
- [ ] 没有堆砌模型式金句
- [ ] 没有反复解释读者已经看懂的主题
- [ ] 情绪通过动作、对话和感官细节呈现
- [ ] 无功能场景、重复问答和重复感悟已删除

## 七、篇幅处理

- 超出 ≤10%：局部压缩后通过。
- 超出 10%—25%：必须压缩重复、解释和无功能场景。
- 超出 >25%：不得直接定稿，向 GLM 提交压缩任务。
- 不得以“保留气口、轻喜节奏好”为理由取消字数硬约束。

## 八、是否退回 GLM

出现任一项，停止 Codex 重写，修 prompt 后重新调用 GLM：

- [ ] 核心问题由配角解决
- [ ] 出现两条无法靠删减分离的主线
- [ ] 主角高光被男主或系统抢走
- [ ] 关系推进方向错误
- [ ] 暗线答案提前说破
- [ ] 整体文风错误
- [ ] 超过 10% 正文需要实质重写
- [ ] 新增三个以上未定案机制或异象

## 九、正式落盘

- [ ] 正式章节不含 prompt、审稿说明、模型自评、修改批注
- [ ] raw 初稿保留在临时目录
- [ ] 审稿报告单独保存
- [ ] 已更新 CLAUDE.md 或进度文件
- [ ] 已更新人物状态
- [ ] 已更新专名表与伏笔表
- [ ] 已将本章暴露的问题写入下一章 prompt 的禁止事项

# 优秀章节 Prompt 设计规则

生成新章节 prompt 前，先读取最接近本章类型的优秀范例。学习结构和决策方式，不复制具体剧情。

## 人物对手戏型

重点学习：

- 开头先定义章节功能和情绪尺度
- 铁律格式置顶
- 只写本章相关人物
- 把抽象机制拆成可观察现象
- 明确双方各自目标、潜台词和关系转折
- 用目标、冲突、转折、钩子、伏笔组织细纲
- 限制无关人物出场和多线展开

## 事件链解决型

重点学习：

- 把抽象主题转化为一个具体乱子
- 乱子原因能追溯到前文改变
- 先写完整因果链，再交给 GLM
- 主角用既有人设能力独立解决
- 配角只能提供有限信息，不能替主角给出答案
- 章末伏笔由当前事件自然触发
- 只种疑，不解释答案

## Prompt 必须具备的九层结构

1. 写手身份、题材、章节功能、情绪尺度
2. 铁律格式
3. 本书最小必要背景
4. 本章相关暗线逻辑和禁词
5. 前情衔接
6. 本章相关人物的表演规则
7. 抽象机制的可观察表现
8. 目标、冲突、转折、钩子、伏笔
9. 连贯、场景、篇幅、手法和输出约束

## 抽象要求的改写方法

错误：

- 展现光环失灵
- 展现蝴蝶效应
- 展现世界正在偏移

正确：

- 主角能自然与光环角色对视，不再自惭形秽
- 前任倒台后，依赖其维持的补拨流程断裂，导致具体资源断供
- 某个原本顺畅的流程因关键人物缺位而卡住，逐级影响现场

抽象机制只给 2—4 个可写表现，并要求 GLM 选一两个。不得全部堆入正文。

## 禁止错误模仿

不得为了显得丰富而：

- 同时添加两条乱子
- 增加新的天命转接、光环转移、能量转移规则
- 同时调用石叶、泪痣、任务牌、灵田、系统等多个异象
- 让男主远程提供完整解决方案
- 让系统直接解释世界规则
- 把下一章的真相提前写入本章
- 为轻喜而连续复用同一个梗
- 因为“气口好”就无视篇幅硬约束

## Claude Code 风格的关键

不是句子更华丽，而是：

- 先判断本章只负责什么
- 只给 GLM 当前章必要信息
- 把抽象主题翻译成可拍摄的事件
- 明确人物认知边界和禁词
- 按五要素验收
- 发现疑点再定向读取设定
- 只修硬伤，不和 GLM 争夺正文作者权
